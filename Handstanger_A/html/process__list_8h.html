◆ add_entries()

bool add_entries ( ProcessList *  pl,
struct ProcessListEntry *  e[],
int  n 
)

Fügt n Einträge zur Prozessliste hinzu. Wenn die Prozessliste nicht alle n Einträge hinzfügen kann, weil sie sonst mehr als MAX_LIST_ENTRIES Einträge fassen müsste, wird keiner der n Einträge hinzugefügt und add_entries gibt False zurück.

Parameters
*plPointer auf die Prozessliste.
*eVektor von Pointern auf Einträge, der zu pl hinzugefügt werden sollen.
nAnzahl der Einträge in e.
Warning
Wenn e weniger als n Einträge enthält kann die Funktion mit einem Speicherfehler abstürzen.
Returns
True wenn das Hinzfügen erfolgreich war, False sonst.
Remarks
Die Funktion überprüft nicht auf doppelte Einträge, da davon ausgegangen wird, dass das Betriebssystem nur Prozesse mit einer eindeutigen pid erzeugt.
See also

◆ get_bottom_n_processes_cpu()

int get_bottom_n_processes_cpu ( const ProcessList *  pl,
int  n,
struct ProcessListEntry *  result_list[] 
)

Sucht die n Prozesse mit der niedrigsten aktuellen Prozessorzeit. Wenn mehrere Prozesse das Suchkriterium erfüllen, werden sie in der Reihenfolge, in der sie in der Prozessliste gespeichert sind zurückgegeben.

Parameters
*plPointer auf die Prozessliste.
nAnzahl der Prozesse, die gesucht werden.
*result_listPointer auf die Liste, welche die Ergebnisse der Suche hält.
Returns
Anzahl der gefundenen Prozesse.
Remarks
Wenn n größer als die Anzahl der Prozesse in pl ist, werden nur MAX_LIST_ENTRIES Einträge in result_list gespeichert und der Rückgabewert ist MAX_LIST_ENTRIES.
Warning
Es wird nicht überprüft, ob result_list ausreichend dimensioniert ist. Falls die Größe von result_list kleiner als n ist, kann die Funktion mit einem Speicherfehler abstürzen.
